Build a production-grade enterprise Question Bank Web Application.

The system must allow administrators to create Subjects, Chapters, Questions, and Choices with LaTeX support, image uploads stored directly inside Microsoft SQL Server as BLOB, live preview rendering, and professional PDF export functionality.

Use clean architecture and scalable modular structure.

TECH STACK REQUIREMENTS

Frontend:

React
TypeScript
TailwindCSS or Bootstrap
KaTeX (for LaTeX rendering)
react-split-pane (for draggable editor preview layout)

Backend:

Node.js
Express.js

Database:

Microsoft SQL Server (MSSQL)

PDF Engine:

Puppeteer

Image Upload Handling:

multer memory storage

Images must be stored inside MSSQL using VARBINARY(MAX), NOT filesystem storage.

APPLICATION FUNCTIONAL FLOW

Step 1:

Create Subjects

Step 2:

Step 3:

Export Questions to PDF

DATABASE DESIGN REQUIREMENTS

Create normalized schema.

Subjects Table

subject_id INT PRIMARY KEY IDENTITY
subject_name NVARCHAR(255)
created_at DATETIME DEFAULT GETDATE()

Chapters Table

chapter_id INT PRIMARY KEY IDENTITY
subject_id INT FOREIGN KEY REFERENCES subjects(subject_id)
chapter_name NVARCHAR(255)
created_at DATETIME DEFAULT GETDATE()

Questions Table

question_id INT PRIMARY KEY IDENTITY
chapter_id INT FOREIGN KEY REFERENCES chapters(chapter_id)

question_text NVARCHAR(MAX)

question_type VARCHAR(20)
Allowed values:

MCQ
FILLUP

difficulty_level VARCHAR(20)
Allowed values:

EASY
MEDIUM
HARD

question_image VARBINARY(MAX)

question_image_name NVARCHAR(255)

question_image_type NVARCHAR(50)

question_image_size INT

created_at DATETIME DEFAULT GETDATE()

Choices Table

choice_id INT PRIMARY KEY IDENTITY
question_id INT FOREIGN KEY REFERENCES questions(question_id)

choice_text NVARCHAR(MAX)

choice_image VARBINARY(MAX)

choice_image_name NVARCHAR(255)

choice_image_type NVARCHAR(50)

choice_image_size INT

is_correct BIT
